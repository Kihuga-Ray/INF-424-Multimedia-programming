Video Transcript & Caption

This tutorial demonstrates advanced video player controls and event handling in HTML5. You'll learn how to create a fully custom video player without using the browser's default controls. The player features real-time state tracking, showing the distinction between 'play' and 'playing' events. The 'play' event fires when you click play but before buffering completes; the 'playing' event fires only when playback actually begins. This video covers playback speed control using the playbackRate property, fullscreen mode with requestFullscreen API, and handling all six major video events: play, playing, pause, waiting, ended, and error. Notice the status panel updates with each state transition, providing insight into the temporal media lifecycle. Keyboard shortcuts include Space for play/pause and F for fullscreen.

Video Event Lifecycle: From Load to Playback

The video player manages six primary events, representing distinct states in the temporal media lifecycle:

State Diagram: Video Lifecycle


[READY] 
  ↓ (user clicks play)
[LOADING] → 'play' event fires
  ↓ (data buffered, playback begins)
[PLAYING] → 'playing' event fires (actual playback started)
  ↓ (user clicks pause OR network stalls)
[PAUSED] or [WAITING] → 'pause' or 'waiting' event fires
  ↓ (playback resumes)
[PLAYING] → 'playing' event fires again
  ↓ (end of media reached)
[ENDED] → 'ended' event fires
▶ 'play' Event

Fires immediately when video.play() is called, BEFORE playback actually begins. May require buffering/loading.

PLAYING ✓ Event

Fires when playback ACTUALLY BEGINS after sufficient buffering. Indicates media is ready for display.

⏸ 'pause' Event

Fires when playback is paused by user action or programmatically via video.pause().

⌛ 'waiting' Event

Fires when playback PAUSES to buffer more data (network lag). Indicates insufficient buffered data.

✓ 'ended' Event

Fires when playback completes and reaches the end of the media timeline. Reset UI controls.

❌ 'error' Event

Fires when media cannot be decoded, network fails, or format is unsupported. Display error message.

Custom Player Features

⏵ Play/Pause Toggle

Uses video.paused property to determine current state and call appropriate method.

📊 Seek Bar

<input type="range"> synced to video.currentTime via timeupdate event.

🎚 Playback Speed

Controlled via video.playbackRate property. Supports 0.75x to 2x speeds.

⛶ Fullscreen

video.requestFullscreen() with cross-browser prefixes. Press ESC to exit.

🔄 Responsive Design

Maintains 16:9 aspect ratio using padding-bottom percentage trick. Works on all screens.
