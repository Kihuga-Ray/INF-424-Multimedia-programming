Audio Event Lifecycle

The audio player handles the following HTML5 media events:

📥 loadstart

Fires when browser starts loading the audio resource.

✓ loadeddata

Fires when current playframe has loaded (enables seeking).

▶ play

Fires when playback requests initiate (may wait for data).

⏸ pause

Fires when playback is paused or stopped.

⏱ timeupdate

Fires ~250ms intervals. Updates UI with current time.

⌛ waiting

Fires when playback pauses while buffering (network stall).

✓ ended

Fires when playback completes. Reset UI to initial state.

❌ error

Fires on decode error, missing file, or unsupported format.
